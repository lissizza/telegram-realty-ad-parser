### How to Contribute

1. **Fork the repository**
2. **Create a new branch** for your feature: `git checkout -b feature/your-feature-name`
3. **Make your changes** following our coding standards
4. **Test your changes** thoroughly
5. **Commit your changes**: `git commit -m "feat: add your feature"`
6. **Push to your fork**: `git push origin feature/your-feature-name`
7. **Create a Pull Request**

### Coding Standards

- Follow PEP8 for Python code
- Use type hints
- Write docstrings for classes and methods
- Use lazy % formatting in logging: `logger.info("User %s logged in", username)`
- Maximum line length: 120 characters
- Chat language: Russian
- Code language: English (comments, variables, documentation)

- Keep PRs focused on a single feature or fix
- Write clear, descriptive commit messages
- Update documentation if needed
- Add tests for new features
- Ensure all tests pass before submitting
- Never commit directly to `main`; use a feature branch and merge via PR
